# 测试

项目中有两种测试：一个是功能测试，一个是集成测试。

- 功能测试：保证函数正确执行，例如访问令牌的签发与验证、密码存储hash的计算与验证。功能测试代码通常和功能代码在一起
- 集成测试：保证接口正确运行，通过构造http请求的方式模拟正式使用场景。代码在单独目录`server/test/`

暂未实现输入测试、并发测试、安全测试等内容

- 输入测试：用户名输入框，输入空白、SQL注入、超长文本等
    - 了解go fuzzing test
- 并发测试：使用k6或类似工具进行并发测试
- 安全测试：权限控制、敏感信息检查
- 可用性测试：操作逻辑是否可以优化，例如3步操作变成2步、填写表单是否保存草稿

## 集成测试

我们为服务端程序提供的接口编写了集成测试：

- 每个接口有数量不等的测试用例
- 每个测试用例通过向服务端程序发送http请求来模拟外部调用
- 每个测试用例都会检查执行结果：异常用例检查错误信息、正常用例检查执行结果
- 集成测试在测试环境中运行、单独构造测试数据
- 测试用例的副作用将被继承，即每一个测试用例除了考虑最开始构造的测试数据，还要考虑之前用例的副作用
    - 也可以为每一个用例构造全套参数，看哪一种更简单、清晰

集成测试通过，说明接口基本符合预期。

## 使用UI测试

使用UI测试每个功能的运行情况与展示效果：

- 集成测试是一键测试，通常在十几/几十秒内完成；使用UI测试则是手动测试，需要根据用例描述执行动作并检查程序响应
- 使用UI测试在初始化的开发环境中运行、使用初始化时的测试数据
