# 命令介绍

`pkill -f unnamed_plan_server -SIGUSR1`
`pkill -f unnamed_plan_server -SIGUSR2`

- `pkill`: `ps | grep | kill`
- `-f`: 给出的匹配字符串长度超过15，需要提供该参数强制执行
- `kill`的含义是向进程发送信号，只是默认发送结束信号。
    - `kill -l`可以查看能发送的信号
    - 信号尽量使用名字而非数字，因为同一个信号在不同架构中可能对应不同的数字
    - 自定义信号和实时信号需要程序编写响应对应信号的处理逻辑
    - 本项目中，响应自定义信号1的代码功能为：手动备份；响应自定义信号2的代码功能为：恢复
- 恢复功能执行步骤：
    - 在老服务器上执行备份
    - 下载老服务器的`backup`文件夹
    - 将`backup`文件夹改名为`recover`
    - 上传`recover`文件夹到新服务器（应与服务端程序在相同路径）
    - 执行恢复
